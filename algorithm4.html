Explanation of the Algorithm

The Exact algorithm works by finding the densest subgraph in an undirected graph G using a flow network. Initially, the algorithm sets up a binary search range based on the degree of vertices and iteratively refines the search using a minimum cut approach. The algorithm involves creating a flow network where each vertex has edges connecting it to source and sink nodes, and clique structures are handled with infinite capacity edges. The main goal is to identify the densest subgraph containing all (h-1)-cliques. It refines its approximation based on st-cut results, adjusting the subgraph and its density calculation at each step.

Pseudocode: CoreExact Algorithm

Algorithm 4: The algorithm: CoreExact.
Input: G(V, E), Ψ(VΨ, EΨ);
Output: The CDS D(VD, ED);

1 perform core decomposition using Algorithm 3;
2 locate the (k'', Ψ)-core using pruning criteria;
3 initialize C ← ∅, D ← ∅, U ← ∅, l ← ρ'', u ← kmax;
4 put all the connected components of (k'', Ψ)-core into C;
5 for each connected component C(VC, EC) ∈ C do
6  if |C|>k'' then C(VC, EC) ← C ∩ ([I], Ψ)-core;
7  build a flow network F(VF, EF) by lines 5-15 of Algorithm 1;
8  find minimum st-cut (S, T) from F(VF, EF);
9  if S=∅ then continue;
10 while u − l ≥ 1 / |VC|(|VC| − 1) do
11    α ← (l + u) / 2;
12    build F(VF, EF) by lines 5-15 of Algorithm 1;
13    find minimum st-cut (S, T) from F(VF, EF);
14    if S = {s} then u ← α;
15    else if α> [I] then remove some vertices from C;
16    l ← α;
17    U ← S\{s};
18 if ρ(G[U], Ψ) > ρ(D, Ψ) then D ← G[U];
19 return D;

Explanation of the Algorithm

The CoreExact algorithm extends the Exact algorithm by incorporating core decomposition. It uses pruning criteria to locate the (k'', Ψ)-core of a graph, then iteratively refines the densest subgraph. The algorithm applies flow network techniques, similar to Exact, and uses a minimum st-cut approach to find the subgraph with the highest density. It compares different subgraph results based on the density values and returns the densest one.
